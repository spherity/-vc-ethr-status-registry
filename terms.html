The following terms are used to describe concepts in this specification.

delegate
An Ethereum address that received temporary access to a revocation lists in a namespace. It has to be granted by the current owner of the revocation list in question. Delegates can change statuses of credentials in foreign revocation lists.
namespace
A namespace is a representation of an Ethereum address inside the registry that corresponds to its owners address. All revocation lists within a namespace are initially owned by the namespaceâ€™s owner address.
owner
An Ethereum address that has modifying rights to revocation lists within its own and possibly foreign namespaces. An owner can give up modifying rights of revocation lists within its namespace by transferring ownership to another address, or invite other addresses as delegates.
registry
A registry is a deployed smart contract containing the code for EthrRevocationRegistry2022. It is the central point of interaction with namespaces and revocation lists that are stored in it and offers a suite of management features according to EIP-5539.
revocation list
A namespace can contain a number of revocation lists. Each revocation list is identified by a unique key of the type bytes32 that can be used to address it in combination with the namespace address.
revocation key
A revocation list can contain a number of revocation keys of the type bytes32. In combination with the namespace address and the revocation list key, it resolves to a boolean value that indicates whether the revocation key is revoked or not.
